Documentação Internuz

Introdução

Bem-vindo à documentação do Projeto Internuz, uma plataforma dinâmica projetada para centralizar os canais de comunicação interno e a conexão entre colaboradores. Desenvolvido com a combinação de AngularJS no frontend e Node.js no backend, juntamente com o banco de dados MongoDB, o Internuz é uma solução versátil e escalável para atender às demandas internas da Nairuz.

Nesta documentação, você encontrará informações detalhadas sobre a arquitetura, funcionalidades e uso do Projeto Internuz. Desde a configuração inicial até a implementação e manutenção para orientar os desenvolvedores e administradores.

Acessos

O código-fonte e repositório de todo projeto está disponível no GitHub da Nairuz. Para realizar manutenção no projeto, insira seu perfil do GitHub como colaborador do projeto e faça o clone de todo repositório para seu ambiente local.

Primeiros Passos

Rodando o projeto localmente

Após clonar o repositório, navegue até o diretório do projeto e instale as dependências tanto para o frontend quanto para o backend. Você pode fazer isso utilizando o npm ou o Yarn. Abra o terminal e execute um dos seguintes comandos:

npm install

ou

yarn install

Com as dependências instaladas com sucesso, você está pronto para iniciar o servidor localmente. O Projeto Internuz utiliza o npm script dev para iniciar tanto o servidor backend quanto o frontend simultaneamente. No terminal, execute o seguinte comando:

npm run dev

Este comando irá iniciar o servidor backend na porta 3000 e o servidor frontend na porta 4200.

Uma vez que ambos os servidores estejam em execução, você pode acessar a aplicação em desenvolvimento digitando http://localhost:4200 em seu navegador web.

Enviando um commit para o repositório

Após concluir as alterações necessárias no Projeto Internuz e validar que tudo está funcionando conforme o esperado, é hora de realizar o envio das atualizações para o repositório no GitHub. Siga os passos abaixo:

Importante: Antes de atualizar o repositório, é necessário buildar o projeto para validar se não é exibido nenhum erro durante o build. Execute o seguinte comando:

ng build

Uma vez que o build do projeto foi concluído com sucesso, é hora de fazer um commit das alterações para o repositório no GitHub. Primeiro, adicione todas as alterações ao índice de preparação executando o comando:

git add .

Em seguida, faça o commit das alterações utilizando o comando git commit. Certifique-se de fornecer uma mensagem descritiva para o commit, explicando as alterações realizadas.

git commit -m "Descrição do que foi realizado"

Finalmente, envie as alterações para o repositório no GitHub utilizando o comando git push. Isso garantirá que as alterações sejam sincronizadas com o repositório remoto.

git push origin master

Validando Deploy

Após enviar as alterações para o repositório, o deploy automático ocorre geralmente entre 5 a 10 minutos. Após esse período, acesse o site em produção (https://internuz.com.br) para validar se as alterações foram refletidas conforme esperado.

Em caso de qualquer problema, verifique na hospedagem do Google Cloud se a nova versão foi implantada corretamente ou se ocorreu alguma falha durante o processo. Isso ajudará a identificar e resolver qualquer questão que possa ter surgido durante o deploy.
